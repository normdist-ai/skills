# PDCA循环方法论技能

## 概述

基于ISO9000质量管理体系的PDCA循环理论，指导代理使用Plan（策划）、Do（实施）、Check（检查）、Act（处置）四阶段帮助用户实现持续改进。

## Gotchas

- **PDCA不是一次性流程** — 必须形成闭环，Act阶段结束后进入下一循环
- **Check阶段必须基于数据** — 不能主观臆断，需要客观数据支撑
- **Plan阶段目标必须SMART** — 具体、可衡量、可实现、相关、有时限
- **5WHY需要追问到底** — 至少连续追问5次，找到根本原因而非表面原因
- **敏捷开发与PDCA兼容** — Sprint本身就是小型PDCA循环

## PDCA四阶段速查

| 阶段 | 核心目标 | 关键工具 | 主要输出 |
|------|----------|----------|----------|
| **Plan** | 建立目标和过程 | SMART、WBS、SIPOC | 目标说明书、WBS、流程图 |
| **Do** | 执行策划方案 | 任务执行、进度跟踪 | 执行记录、进度报告 |
| **Check** | 监视和测量结果 | 5WHY、偏差分析 | 检查报告、根因记录 |
| **Act** | 持续改进绩效 | 标准化、经验总结 | 标准化文件、改进计划 |

## 工作流程 Checklist

Progress:
- [ ] 1. Plan（策划）阶段
  - [ ] 1.1 识别问题或改进机会
  - [ ] 1.2 设定SMART目标
  - [ ] 1.3 进行SIPOC过程分析
  - [ ] 1.4 制定WBS工作分解
  - [ ] 1.5 识别资源和风险
- [ ] 2. Do（实施）阶段
  - [ ] 2.1 按WBS执行任务
  - [ ] 2.2 记录执行过程
  - [ ] 2.3 监控进度和偏差
  - [ ] 2.4 收集证据材料
- [ ] 3. Check（检查）阶段
  - [ ] 3.1 对照目标验证结果
  - [ ] 3.2 进行偏差分析
  - [ ] 3.3 执行5WHY根因分析
  - [ ] 3.4 形成检查报告
- [ ] 4. Act（处置）阶段
  - [ ] 4.1 提炼成功经验
  - [ ] 4.2 记录失败教训
  - [ ] 4.3 进行标准化
  - [ ] 4.4 制定下一循环改进计划

## 验证循环

### 步骤1: 验证Plan阶段完整性

检查以下输出物是否齐全：
- 目标说明书（含SMART验证）
- WBS工作分解结构
- SIPOC过程分析
- 资源需求计划

如不完整，返回补充后再进入Do阶段。

### 步骤2: 验证Check阶段数据

检查以下内容：
- 是否有客观数据支撑？
- 目标达成率是否可计算？
- 根因分析是否找到根本原因？

如数据不足，返回Do阶段补充数据收集。

### 步骤3: 验证Act阶段闭环

确认以下内容：
- 是否形成标准化文件？
- 是否设定下一循环目标？
- 是否更新知识库？

只有完成闭环，PDCA循环才算完成。

## 核心工具

### SMART目标设定

| 要素 | 含义 | 检查问题 |
|------|------|----------|
| S | 具体的 | 目标是否明确？ |
| M | 可衡量的 | 是否可以量化？ |
| A | 可实现的 | 是否切实可行？ |
| R | 相关的 | 是否与战略相关？ |
| T | 有时限的 | 是否有明确期限？ |

## 快速开始

1. **识别阶段**：判断用户处于PDCA哪个阶段
2. **提供指导**：给出该阶段的关键活动和输出物
3. **引导推进**：完成当前阶段后，引导进入下一阶段
4. **形成闭环**：确保完成完整的PDCA循环
