# PDCA术语表（中英文对照）

## 核心术语

| 英文 | 中文 | 说明 |
|------|------|------|
| **PDCA Cycle** | PDCA循环 | Plan-Do-Check-Act循环，又称戴明环 |
| **Deming Cycle** | 戴明环 | 以W. Edwards Deming命名的PDCA循环 |
| **Plan** | 策划 | PDCA循环的第一阶段，建立目标和过程 |
| **Do** | 实施 | PDCA循环的第二阶段，执行策划方案 |
| **Check** | 检查 | PDCA循环的第三阶段，监视和测量结果 |
| **Act** | 处置 | PDCA循环的第四阶段，持续改进绩效 |
| **Continuous Improvement** | 持续改进 | 不断改进过程绩效的循环活动 |
| **Closed Loop** | 闭环 | 完整的PDCA循环，确保持续改进 |

## 质量管理术语

| 英文 | 中文 | 说明 |
|------|------|------|
| **Quality Management System (QMS)** | 质量管理体系 | 指导和控制组织的管理体系 |
| **ISO 9000** | ISO 9000标准 | 质量管理体系基础和术语标准 |
| **ISO 9001** | ISO 9001标准 | 质量管理体系要求标准 |
| **Quality Policy** | 质量方针 | 由最高管理者正式发布的质量宗旨和方向 |
| **Quality Objective** | 质量目标 | 在质量方面所追求的目的 |
| **Process Approach** | 过程方法 | 将活动作为过程进行管理的方法 |

## 工具方法术语

### 目标设定工具

| 英文 | 中文 | 说明 |
|------|------|------|
| **SMART Criteria** | SMART原则 | 具体、可衡量、可实现、相关、有时限的目标设定原则 |
| **Specific** | 具体的 | 目标必须明确具体 |
| **Measurable** | 可衡量的 | 目标必须可以量化衡量 |
| **Achievable** | 可实现的 | 目标必须切实可行 |
| **Relevant** | 相关的 | 目标必须与整体战略相关 |
| **Time-bound** | 有时限的 | 目标必须有明确的时间节点 |

### 工作分解工具

| 英文 | 中文 | 说明 |
|------|------|------|
| **Work Breakdown Structure (WBS)** | 工作分解结构 | 将项目分解为可管理工作包的层次结构 |
| **Milestone** | 里程碑 | 项目中的重要节点或事件 |
| **Work Package** | 工作包 | WBS中最低层次的工作单元 |
| **Deliverable** | 交付物 | 为完成项目而产出的可验证成果 |
| **Gantt Chart** | 甘特图 | 显示项目进度的时间条形图 |

### 过程分析工具

| 英文 | 中文 | 说明 |
|------|------|------|
| **SIPOC Model** | SIPOC模型 | 供方-输入-过程-输出-顾客分析模型 |
| **Supplier** | 供方 | 提供输入的组织或个人 |
| **Input** | 输入 | 过程所需的资源或材料 |
| **Process** | 过程 | 将输入转化为输出的活动 |
| **Output** | 输出 | 过程产生的结果 |
| **Customer** | 顾客 | 接受输出的组织或个人 |

### 根因分析工具

| 英文 | 中文 | 说明 |
|------|------|------|
| **5 Whys Analysis** | 5WHY分析法 | 连续追问"为什么"找到根本原因的方法 |
| **Root Cause** | 根本原因 | 导致问题的最基本原因 |
| **Root Cause Analysis (RCA)** | 根因分析 | 识别问题根本原因的系统方法 |
| **Fishbone Diagram** | 鱼骨图 | 又称因果图，用于分析问题原因 |

### 风险分析工具

| 英文 | 中文 | 说明 |
|------|------|------|
| **SWOT Analysis** | SWOT分析 | 优势、劣势、机会、威胁分析 |
| **Strengths** | 优势 | 组织内部的积极因素 |
| **Weaknesses** | 劣势 | 组织内部的消极因素 |
| **Opportunities** | 机会 | 组织外部的积极因素 |
| **Threats** | 威胁 | 组织外部的消极因素 |

### 分类工具

| 英文 | 中文 | 说明 |
|------|------|------|
| **MECE Principle** | MECE原则 | 相互独立、完全穷尽的分类原则 |
| **Mutually Exclusive** | 相互独立 | 分类之间没有重叠 |
| **Collectively Exhaustive** | 完全穷尽 | 分类覆盖所有可能性 |

## 改进术语

| 英文 | 中文 | 说明 |
|------|------|------|
| **Kaizen** | 改善 | 日本的持续改进理念 |
| **Lean** | 精益 | 消除浪费、创造价值的管理方法 |
| **Six Sigma** | 六西格玛 | 减少变异、提高质量的方法论 |
| **Total Quality Management (TQM)** | 全面质量管理 | 全员参与的质量管理方法 |
| **Best Practice** | 最佳实践 | 被证明有效的做法或方法 |
| **Standardization** | 标准化 | 将成功做法制定为标准的过程 |

## 项目管理术语

| 英文 | 中文 | 说明 |
|------|------|------|
| **Project** | 项目 | 为达到特定目标而进行的临时性工作 |
| **Project Management** | 项目管理 | 对项目进行策划、执行、监控的活动 |
| **Project Charter** | 项目章程 | 正式批准项目的文档 |
| **Stakeholder** | 干系人 | 与项目有利害关系的人员或组织 |
| **Scope** | 范围 | 项目的工作边界和内容 |
