# DevLog Skill - 开发工作日志

这是一个用于生成专业软件开发工作日志的 Agent Skill。

## 功能特性

- ✅ **双格式支持**: 标准模板和时间线格式
- ✅ **工作量统计**: 自动计算工作时长、任务数、代码提交等
- ✅ **问题追踪**: 详细记录遇到的问题和解决方案
- ✅ **Git集成**: 自动从Git历史提取提交信息
- ✅ **灵活存储**: 按天分文件夹组织,支持多种日志类型
- ✅ **可扩展**: 支持自定义模板和统计维度
- ✅ **进度跟踪**: TodoList机制实时显示执行进度

## 快速开始

### 1. 安装技能

将 `devlog` 文件夹复制到以下位置之一:

**项目级别** (团队共享):
```bash
cp -r devlog /your-project/.lingma/skills/
```

**个人级别** (所有项目可用):
```bash
cp -r devlog ~/.lingma/skills/
```

### 2. 使用技能

在对话中直接提及即可触发:

```
用户: "帮我写今天的工作日志"
用户: "生成今日日报"
用户: "记录一下今天的工作"
用户: "用时间线格式生成日志"
```

Agent 会自动:
1. 收集工作信息 (从对话或Git历史)
2. 询问或选择输出格式
3. 生成日志内容
4. 保存到 `.devlog/YYYY-MM-DD/` 目录

### 3. 查看日志

日志保存在项目根目录的 `.devlog` 文件夹:

```
.devlog/
├── index.md                    # 总索引
├── 2026-04-30/                 # 按天分文件夹
│   ├── log.md                  # 标准格式日志
│   ├── timeline.md             # 时间线格式日志
│   └── stats.json              # 统计数据
└── 2026-04-29/
    └── log.md
```

## 文件格式

### 标准模板格式

适合正式汇报和详细记录,包含:
- 📋 完成事项
- ⏱️ 时间线
- 🐛 遇到的问题
- 📊 工作量统计
- 📝 明日计划
- 💡 备注

### 时间线格式

适合快速记录和回顾,简洁明了:
- 按时间顺序的关键事件
- 基本统计数据

## 高级用法

### 使用统计脚本

```bash
# 生成今日统计
python .lingma/skills/devlog/scripts/stats.py --date 2026-04-30

# 生成本周统计
python .lingma/skills/devlog/scripts/stats.py --week

# 生成月度统计
python .lingma/skills/devlog/scripts/stats.py --month 2026-04

# 输出文本报告
python .lingma/skills/devlog/scripts/stats.py --date 2026-04-30 --report
```

### 自定义模板

修改 `templates/` 目录下的模板文件:
- `templates/standard.md` - 标准模板
- `templates/timeline.md` - 时间线模板

### Git忽略配置

如果不需要将日志提交到版本控制,在 `.gitignore` 中添加:

```gitignore
# 工作日志
/.devlog/
```

## 最佳实践

1. **及时记录**: 每天工作结束时生成日志
2. **具体明确**: 使用具体的描述和数据
3. **问题导向**: 详细记录问题和解决方案
4. **保持一致**: 统一的术语和格式
5. **定期回顾**: 每周/每月总结工作趋势

详细指南请查看: [references/best-practices.md](references/best-practices.md)

## 常见问题

**Q: 如何修改日志保存位置?**  
A: 修改 `SKILL.md` 中的保存路径,默认是 `.devlog/`。

**Q: 可以同时生成两种格式吗?**  
A: 可以,Agent 会分别保存到 `log.md` 和 `timeline.md`。

**Q: 如何生成周报或月报?**  
A: 使用统计脚本: `python scripts/stats.py --week` 或 `--month YYYY-MM`。

**Q: 没有Git仓库可以使用吗?**  
A: 可以,Git信息是可选的。如果没有Git,Agent会提示手动提供信息。

**Q: 如何自定义统计维度?**  
A: 修改 `scripts/stats.py` 添加新的统计逻辑,或在模板中添加新字段。
