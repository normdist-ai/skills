# TodoList 跟踪使用指南

## 基本用法

DevLog 技能在执行过程中始终维护 TodoList 以跟踪进度。

### 标记说明

| 标记 | 含义 | 使用场景 |
|------|------|---------|
| `- [ ]` | 待处理 | 任务开始前 |
| `- [x]` | 已完成 | 任务成功完成 |
| `- [-]` | 已跳过 | 数据源不可用或任务不适用 |

### 使用规则

1. **开始时**: 复制 SKILL.md 中的 TodoList 模板到回复中
2. **执行中**: 每完成一个子任务,将 `[ ]` 改为 `[x]`
3. **跳过项**: 标记为 `[-]` 并说明原因(如"非Git仓库,跳过Git数据源")
4. **结束时**: 展示完整的 TodoList,所有项应为 `[x]` 或 `[-]`

### 示例

```
任务进度跟踪:
- [x] 步骤1: 多数据源信息收集
  - [x] 1.1 分析对话上下文
  - [x] 1.2 提取Git提交历史
  - [-] 1.3 扫描文件系统活动 (非Git仓库,跳过)
  - [-] 1.4 检查终端命令历史 (无历史记录,跳过)
- [x] 步骤2: 确定输出格式
- [x] 步骤3: 智能信息提取和分类
- [x] 步骤4: 生成日志内容
- [x] 步骤5: 保存日志文件
- [x] 步骤6: 确认和反馈
```

## 进度反馈

在步骤6确认和反馈时,必须包含:
1. 完整的 TodoList(所有项标记完成)
2. 数据源使用摘要(哪些可用,哪些跳过)
3. 生成内容摘要(日期/格式/文件列表)
4. 后续操作建议
