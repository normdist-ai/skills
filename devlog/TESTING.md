# DevLog 技能测试指南

本文档提供测试 DevLog 技能的示例场景和预期结果。

## 测试前准备

1. 确保技能已安装到正确位置:
   - 项目级别: `.lingma/skills/devlog/`
   - 或个人级别: `~/.lingma/skills/devlog/`

2. (可选) 初始化Git仓库以测试Git集成功能:
   ```bash
   git init
   git config user.name "Test User"
   git config user.email "test@example.com"
   ```

## 测试场景

### 场景1: 基本日志生成

**输入**:
```
帮我生成今天的工作日志
```

**预期行为**:
1. Agent 询问或从对话中提取工作信息
2. 如果信息不足,主动询问关键内容
3. 默认使用标准模板格式
4. 生成日志并保存到 `.devlog/YYYY-MM-DD/log.md`

**预期输出结构**:
```markdown
# 工作日志 - 2026-04-30

## 📋 完成事项
- [x] ...

## ⏱️ 时间线
- ...

## 🐛 遇到的问题
- ...

## 📊 工作量统计
- ...

## 📝 明日计划
- ...

## 💡 备注
- ...
```

---

### 场景2: 指定格式

**输入**:
```
用时间线格式记录今天的工作
```

**预期行为**:
1. 使用时间线模板
2. 生成简洁的时间线日志
3. 保存到 `.devlog/YYYY-MM-DD/timeline.md`

**预期输出结构**:
```markdown
# 工作日志 - 2026-04-30

## 时间线
- **[HH:MM]** ...

## 统计
- 总时长: X小时
- 任务数: N个
```

---

### 场景3: 提供详细工作内容

**输入**:
```
今天我完成了以下工作:
1. 上午修复了登录bug,主要是session过期的问题,花了2小时
2. 下午开发了用户管理模块的CRUD接口
3. 遇到了数据库连接池的问题,最后把配置从10改到50解决了
明天计划完善权限控制功能
```

**预期行为**:
1. 提取所有关键信息
2. 分类整理到对应章节
3. 识别问题和解决方案
4. 生成完整的标准格式日志

**预期提取结果**:
- 完成事项: 2个任务
- 问题追踪: 1个问题(数据库连接池)
- 时间信息: 上午2小时 + 下午
- 明日计划: 权限控制功能

## 常见问题排查

### 问题1: 技能未触发

**症状**: Agent 没有使用 DevLog 技能

**检查**:
1. 技能是否正确安装到 `.lingma/skills/` 或 `~/.lingma/skills/`
2. `SKILL.md` 的 frontmatter 是否正确
3. description 是否包含触发关键词

**解决**:
```yaml
# 确保 SKILL.md 开头有正确的 frontmatter
---
name: devlog
description: 生成专业的软件开发工作日志...当用户要求记录、生成或更新工作日志时自动触发...
---
```

---

### 问题2: Git信息提取失败

**症状**: 无法获取Git提交信息

**检查**:
1. 是否在Git仓库中运行
2. Git命令是否正确执行

**解决**:
```bash
# 测试Git命令
git log --since="today" --oneline

# 如果失败,初始化Git
git init
git add .
git commit -m "Initial commit"
```

---

### 问题3: 日志内容不完整

**症状**: 生成的日志缺少某些字段

**检查**:
1. 是否提供了足够的输入信息
2. Agent 是否正确解析了信息

**解决**:
- 提供更详细的工作内容描述
- 明确指定需要包含的字段
- 检查模板文件是否完整

---

### 问题4: 文件保存路径错误

**症状**: 日志保存到错误的位置

**检查**:
1. SKILL.md 中的保存路径配置
2. 是否有权限创建目录

**解决**:
- 确认路径为 `.devlog/YYYY-MM-DD/`
- 检查文件系统权限

## 性能测试

### 测试指标

- **响应时间**: 从输入到生成日志的时间 < 10秒
- **准确性**: 信息提取准确率 > 90%
- **完整性**: 必填字段填充率 100%

### 测试方法

1. 准备10个不同的工作场景描述
2. 分别测试标准格式和时间线格式
3. 记录每次的响应时间和结果质量
4. 统计平均性能和准确率
